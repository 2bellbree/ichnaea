ichnaea.models.hashkey¶

Classes representing unique hashable keys and related database query helpers.

class ichnaea.models.hashkey.HashKey(**kw)[source]¶

A class representing a unique combination of fields, much like a namedtuple. Instances of this class can be used as dictionary keys.

__hash__()[source]¶

Returns a hash of a tuple of the instance values in the same order as the _fields definition.

class ichnaea.models.hashkey.HashKeyQueryMixin[source]¶

A database model mixin, where the classes hashkey represents the primary key columns of the model.

classmethod joinkey(key)[source]¶

Create a SQLAlchemy filter criterion for doing a primary key lookup.

classmethod getkey(session, key)[source]¶

Given a database session, returns the row matching the passed in hashkey.

classmethod querykey(session, key)[source]¶

Given a database session, returns a SQLAlchemy query object with a filter set to return the row matching the passed in hashkey.

classmethod iterkeys(session, keys, extra=None)[source]¶

Called with a database session, a list of hashkeys and an optional callable that will be attached to the database queries, for example a load_only or options.

This is an iterator returning all the matching rows, while the underlying database queries are batched into chunks based on the classes _query_batch constant.

Note that providing an order option via extra won’t work, as it would only order each chunked query but won’t provide a total order over all returned rows.
